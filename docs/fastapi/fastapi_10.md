# 第三部分：制作网站

第二部分 是对 FastAPI 的快速浏览，让您能够迅速上手。这部分将更广泛和深入地讨论细节。我们将构建一个中型网络服务，用于访问和管理关于神秘生物（虚构的生物）和同样虚构的探险家的数据。

正如我之前所述，整个服务将有三个层次：

网络

网络界面

服务

业务逻辑

数据

整个系统的宝贵 DNA

除此之外，网络服务还将具备以下跨层组件：

模型

Pydantic 数据定义

测试

单元、集成和端到端测试

网站设计将解决以下问题：

+   每个层次应包含哪些内容？

+   信息是如何在各层之间传递的？

+   我们是否可以在不破坏任何东西的情况下修改/添加/删除代码？

+   如果出现问题，如何查找并修复？

+   安全问题如何处理？

+   网站能够扩展并表现良好吗？

+   我们能够保持一切尽可能清晰简单吗？

+   为什么我问这么多问题？为什么啊，为什么？
